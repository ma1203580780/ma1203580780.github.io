调试用到命令：

ps -ef |grep php

./php71-fpm reload

cd /usr/local/etc/php/7.1/php-fpm.d/

vi www.conf

Nginx.conf

可以拆分成sites-available和sites-enabled目录，然后available通过软连接到enabled，
在nginx.conf文件中，include进来enabled这个目录下的文件。

看来必须要学习下shell脚本语言了。

可以安装多个php版本，如Mac自带的php56，用brew安装的php70，php71，安装nginx的话，还需要安装php-fpm，这里nginx需要在server中include绑定不同的php-fpm。

php-fpm

php-fpm是mac下自带的软件, 而且兼容不同的PHP版本, 不用额外安装, 但是fpm是需要配置的, 在/private/etc下有个模板, 执行

cd /usr/local/etc/php/7.1/php-fpm.d
cp www.conf

修改php-fpm.conf中的listen行,

将listen = 127.0.0.1改为listen = /tmp/php-fpm.sock,
这样运行php-fpm后就会生成/tmp/php-fpm.sock文件,
nginx可以与这个socket文件交互了(当然也可以用默认的监听端口的方式,
但是指定socket的方式可以避免端口占用的问题, 我认为要放心一些)
修改error_log项, 改为error_log = /usr/local/var/log/php-fpm.log
修改pid项, 改为pid = /usr/local/var/run/php-fpm.pid, pid文件记录fpm的pid, 后续可以用cat /usr/local/var/run/php-fpm.pid的方式获得pid
sudo php-fpm, 大功告成!

注意：不要再root权限下安装东西。

几个关键的目录：

/usr/local/sbin : php-fpm.conf

/usr/local/etc/php/7.1/php-fpm.d : www.conf
